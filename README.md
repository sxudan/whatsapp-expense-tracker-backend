## Description

A WhatsApp webhook application for tracking expenses using natural language. Chat with the bot like you're talking to a friend, and it will automatically parse and save your expenses using OpenAI.

- ðŸ‘¤ Automatic user creation by phone number
- ðŸ“Š REST API endpoints for expense queries

### How it works

1. Send a WhatsApp message with an expense (e.g., "Spent $50 on lunch" or "Coffee $5")
2. The app uses OpenAI to parse the expense details (amount, description, category, date)
3. The expense is saved to the database
4. You receive a confirmation message back via WhatsApp

## Environment Variables

Create a `.env` file in the root directory with the following variables:

```env
# Server
PORT=3000

# Database
DB_HOST=localhost
DB_PORT=5432
DB_NAME=expense_tracker
DB_USER=postgres
DB_PASSWORD=your_password

# WhatsApp API
WHATSAPP_VERIFY_TOKEN=your_verify_token
WHATSAPP_ACCESS_TOKEN=your_access_token
WHATSAPP_BASE_API_URL=https://graph.facebook.com
WHATSAPP_API_VERSION=21.0

# OpenAI
OPENAI_API_KEY=your_openai_api_key
OPENAI_MODEL=gpt-3.5-turbo
```

## Database Setup and Migrations

Before starting the application, you need to set up the database and run migrations:

```bash
# Generate initial migration (first time only)
# Note: Use -- to pass arguments to the script
npm run migration:generate -- src/db/migrations/InitialMigration -d src/db/data-source.ts

# Run migrations
npm run migration:run

# Check migration status
npm run migration:show

# Revert last migration (if needed)
npm run migration:revert
```

### Migration Scripts

- `npm run migration:generate` - Generate a new migration based on entity changes
- `npm run migration:create` - Create an empty migration file
- `npm run migration:run` - Run all pending migrations
- `npm run migration:revert` - Revert the last migration
- `npm run migration:show` - Show migration status

**Important:** Always run migrations before starting the application for the first time.
